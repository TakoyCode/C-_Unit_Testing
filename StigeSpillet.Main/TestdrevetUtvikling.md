# Testdrevet utvikling / Test-driven development

Kan drive med unit-testing uten TDD, men kan ikke gjøre TDD uten unit-testing

Test drevet utvikling handler om at vi skal avlaste hjernen med alle bekymringe som kan feile.
Bekymringene skriver vi ut som unit tester, og når vi implementerer dem så gjør vi det på aller enkleste vis.
